NIGRL — Branching Skill Tree System
=====================================
Design Document v1.0 | 2026-03-23
Inspired by Bloons Tower Defense 6's upgrade commitment system.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
I. CONCEPT SUMMARY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Skills currently progress linearly from levels 1-10, with each level granting
a single perk. This redesign replaces the back half of each skill tree with a
branching commitment system: levels 1-3 remain a shared trunk of foundational
perks, then at level 4 the skill forks into 2-3 specialized branches. Players
commit to one main branch (all tiers) and may dip one tier into a second branch.
Everything else locks out.

This creates meaningful build identity. A player who invests in Smoking will be
either a Cloud Chaser who nukes rooms with Phat Cloud damage or a Contact High
utility player who smokes for debuffs and invisibility. Both identify as
"Smoking builds" but their moment-to-moment gameplay diverges sharply.

The system is designed to pilot on 3-5 skills. Skills not yet redesigned into
branches continue to use their current linear list format — the code must
handle both formats transparently.

II. COMMITMENT RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TRUNK (Levels 1-3): Shared, non-optional. Every player who levels the skill
  gets all trunk perks in order. These are stat boosts and entry-level passives
  that define the skill's identity without locking in a playstyle.

BRANCH SPLIT (Level 4+): At level 4, the player chooses which branch to invest
  in when spending skill points. Branches have ~3-4 tiers (T1-T3 or T1-T4).
  Spending into any branch tier costs skill points the same as any perk level.

COMMITMENT RULE (BTD 5-2-0 adapted):
  - One MAIN branch: can go all the way to T3 or T4.
  - One DIP branch: may invest in T1 only.
  - All other branches: locked out entirely.
  - Lock-in point: once any branch reaches T2, that branch becomes the MAIN.
    You may still dip T1 into one other branch. You can never go T2+ in a
    second branch after committing.
  - T1 is the "taste test" — spend it before committing to see what a branch
    is about. Once you add T2 to any branch, the build is declared.

IMPORTANT: The player can invest T1 in one branch and T1 in a SECOND branch
before committing to neither. Once they spend T2 into a branch, that becomes
the main and the other branch's T1 dip is locked there. No third branch is
ever accessible.

EXAMPLE STATES:
  cloud_chaser: 0, contact_high: 0 → no main declared, all dips available
  cloud_chaser: 1, contact_high: 0 → no main yet; can dip contact_high T1 or
                                      commit to cloud_chaser T2
  cloud_chaser: 1, contact_high: 1 → no main yet; must commit T2 to one
  cloud_chaser: 2, contact_high: 1 → MAIN = cloud_chaser; contact_high dip
                                      locked at T1, no other branches
  cloud_chaser: 3, contact_high: 1 → max main depth; dip locked

III. XP AND SKILL POINTS — NO CHANGE TO ACQUISITION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

XP earning is completely unchanged. You earn "Smoking XP" by smoking joints.
Potential XP converts to real XP via the global skill_points pool. The existing
bksmt_mod formula and DEFAULT_EXP_CURVE both remain in effect.

The only change is WHAT you are spending skill points ON. Instead of advancing
a flat level 1-10, you are advancing either the trunk (levels 1-3) or a specific
branch tier (T1, T2, T3). Internally, the Skill class tracks a single integer
for total levels spent, but also tracks branch_levels per branch. The XP curve
still applies per-level in sequence — trunk L1/L2/L3 use thresholds 0→3, then
each branch tier counts as the next threshold in the same curve.

The existing thresholds:
  DEFAULT_EXP_CURVE = [200, 400, 600, 800, 2000, 6000, 15000, 25000, 100000, 500000]

Trunk levels use curve[0], curve[1], curve[2] (cheapest).
Branch T1 uses curve[3]. T2 uses curve[4]. T3 uses curve[5].
This means the first branch commitment is significantly more expensive than trunk
levels, which is intentional — branches are late-game investments.

A player who dips two branches (T1 each) pays curve[3] twice, total 1600 SP
to get both dips, then faces curve[4] to commit their main.

IV. SMOKING SKILL — PILOT EXAMPLE TREE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

NOTE: The existing Smoking L1-5 are referenced here. The new design replaces
  L4+ with the branching structure. L1-3 become the trunk unchanged except
  for minor flavor tightening. Existing L4 "Roach Fiend" and L5 "Contact High"
  are redistributed into branches.

CURRENT SMOKING (from perk-reference-table.txt and skills.py):
  L1: +2 TOL
  L2: +2 TOL, +2 CON
  L3: Phat Cloud (passive) — deals 10 + (tol // 2) dmg to nearest visible enemy on smoke
  L4: Roach Fiend — 30% chance joint not consumed when smoked
  L5: Contact High — strain effects spread to all enemies within 3 tiles

NEW SMOKING STRUCTURE:
  Trunk L1: +2 TOL [stat]
  Trunk L2: +2 TOL, +2 CON [stat]
  Trunk L3: Phat Cloud [passive] — deals 10 + (tol // 2) dmg to nearest visible enemy on smoke

  (Branch split after trunk L3)

  ─── BRANCH A: "Cloud Chaser" (damage) ───────────────────────────────────────

  Theme: Phat Cloud becomes a primary offensive tool, scaling with Tolerance.
  XP fantasy: the player who pumps Tolerance high and turns every smoke session
  into a room-clearing AoE event.

  T1 — Thick Cloud [passive]
    Phat Cloud now hits 2 nearest visible enemies instead of 1. Targeting
    priority: pick 2 visible enemies nearest to the player by tile distance.
    If only 1 visible enemy exists, behaves as before.
    Cost: curve[3] = 800 SP

  T2 — Hotbox [passive]
    Phat Cloud hits ALL visible enemies within 4 tiles.
    The 4-tile radius is computed as Chebyshev distance (standard).
    This replaces the "2 nearest" rule from Thick Cloud — Thick Cloud's
    selection logic is simply superseded. If you have Hotbox, all visible
    enemies within 4 tiles are hit.
    Cost: curve[4] = 2000 SP

  T3 — Scorched Lungs [passive]
    Phat Cloud damage formula upgrades to: 15 + tolerance (roughly 2x base at
    typical tolerance values). Additionally, enemies at or below 10% of their
    max HP when struck by Phat Cloud are instantly killed (no defense roll).
    The 10% execution threshold is calculated from entity max_hp field.
    Cost: curve[5] = 6000 SP

  ─── BRANCH B: "Contact High" (utility/debuff) ───────────────────────────────

  Theme: Smoking becomes a tactical utility tool. Less raw damage, more fight
  setup. The player uses smoke to debuff enemies before or during engagements.
  Roach Fiend (previously L4) becomes the T1 dip here so resource economy is
  accessible even on the "taste test" tier.

  T1 — Roach Fiend [passive]
    30% chance a joint is not consumed when smoked. This is the existing L4
    perk migrated into the branch structure.
    Cost: curve[3] = 800 SP

  T2 — Secondhand Smoke [passive]
    Smoking applies a 2-turn slow (modified_energy_gain = -25 energy/tick) to
    all enemies within 3 tiles of the player at the moment of smoking.
    Uses Chebyshev radius 3. Does not require line-of-sight to slow enemies —
    the smoke fills the area. Useful for pre-engagement setup.
    New effect needed: "secondhand_slow" (duration=2, modify_energy_gain=-25)
    Cost: curve[4] = 2000 SP

  T3 — Smoke Screen [activated/passive]
    Smoking also grants the player 3 turns of invisibility. While invisible,
    enemies lose sight of the player — any enemy that had the player in their
    CHASING/CHASING state transitions to WANDERING, and the player does not
    trigger sight detection. Invisibility breaks on player attacking or using an
    ability. If the player attacks while invisible, the strike is guaranteed to
    land (no accuracy roll), then invisibility ends.
    New effect needed: "smoke_screen_invis" (duration=3, breaks on melee/ability
    attack, CHASING enemies transition to WANDERING on apply)
    Cost: curve[5] = 6000 SP

  Note: Contact High (existing L5, "strain effects spread within 3 tiles")
  becomes an OPTIONAL third branch called "Connoisseur" in the extended design
  below. If only 2 branches are piloted, Contact High is dropped or folded into
  Branch B T3 slot (replacing Smoke Screen with a combined effect).

  ─── BRANCH C (OPTIONAL): "Connoisseur" (strain synergy) ─────────────────────

  Theme: For players who invest heavily in strain crafting. Amplifies the
  effects of strain joints and creates synergy with Rolling skill's strain
  interactions.

  T1 — Roll Your Own [passive]
    When you smoke a joint with any strain applied, +1 Tolerance for the rest
    of the floor. Stacks up to 3 times per floor. Pairs with Rolling's strain
    perks.
    Cost: curve[3] = 800 SP

  T2 — Contact High [passive]
    Strain effects from the smoked joint spread to all enemies within 3 tiles.
    This is the existing L5 perk migrated here. Each enemy gets a fresh copy of
    the strain effect at the same duration/intensity as the player's joint would
    apply.
    Cost: curve[4] = 2000 SP

  T3 — Chemist [passive]
    When Phat Cloud is triggered, the cloud also carries the strain effect of the
    joint smoked (if any) and applies it to each enemy hit. Requires Trunk L3
    Phat Cloud. Synergizes with Cloud Chaser dip (Thick Cloud gets strain spread
    + damage). The strain application uses the same enemy-applies-strain logic
    used elsewhere for strain items.
    Cost: curve[5] = 6000 SP


EXAMPLE BUILD PATHS — SMOKING:

  Cloud Chaser Full (8 levels total):
    Trunk L1, L2, L3 → Branch A T1, T2, T3 (main)
    Result: every smoke session deals 15+tol to all visible enemies in 4 tiles,
    executes any below 10% HP. Room-nuke build.

  Cloud Chaser + Roach Fiend Dip (9 levels total):
    Trunk L1, L2, L3 → Branch A T1, T2, T3 + Branch B T1 dip
    Result: as above, plus 30% chance not to consume the joint. Resources last
    longer, sustaining the room-nuke.

  Contact High Full (8 levels total):
    Trunk L1, L2, L3 → Branch B T1, T2, T3 (main)
    Result: joints aren't consumed 30% of the time, smoking slows all nearby
    enemies for 2 turns, then the smoke grants invisibility for 3 turns. Stealth
    utility build.

  Contact High + Thick Cloud Dip (9 levels total):
    Trunk L1, L2, L3 → Branch B T1, T2, T3 + Branch A T1 dip
    Result: utility-primary with some AoE coverage. 2-target Phat Cloud + slow
    debuff + smoke screen.

  Pure Connoisseur (if 3-branch variant, 8 levels total):
    Trunk L1, L2, L3 → Branch C T1, T2, T3 (main)
    Result: strain specialist. Each smoke of a strained joint boosts Tolerance,
    spreads strain to all nearby enemies, and Phat Cloud carries the strain.
    Best paired with Rolling investment for crafted strain joints.

V. DATA MODEL CHANGES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CURRENT STRUCTURE (skills.py):
  SKILL_PERKS["Smoking"] = [perk_L1, perk_L2, ..., perk_L10]
  A flat list of 10 dicts. Index 0 = level 1, index 9 = level 10.

NEW STRUCTURE (branched skills only):
  SKILL_PERKS["Smoking"] = {
      "trunk": [perk_L1, perk_L2, perk_L3],
      "branches": {
          "cloud_chaser": {
              "name": "Cloud Chaser",
              "perks": [tier1_dict, tier2_dict, tier3_dict],
          },
          "contact_high": {
              "name": "Contact High",
              "perks": [tier1_dict, tier2_dict, tier3_dict],
          },
          "connoisseur": {
              "name": "Connoisseur",
              "perks": [tier1_dict, tier2_dict, tier3_dict],
          },
      },
      "max_main_depth": 3,    # how many tiers the main branch can go
      "max_dip_depth":  1,    # how many tiers a dip branch can go
  }

  Each perk dict in trunk and branches uses the SAME schema as the current
  flat list: {"name": str, "perk_type": str, "effect": dict|None, "desc": str}

DETECTION CONVENTION:
  isinstance(SKILL_PERKS[name], list)  → linear skill, old behavior
  isinstance(SKILL_PERKS[name], dict)  → branching skill, new behavior
  This lets both formats coexist during the pilot rollout without flag fields.

SKILL CLASS CHANGES (skills.py):

  Current Skill class fields:
    self.name: str
    self.level: int          # total levels earned (0-10)
    self.real_exp: float
    self.potential_exp: float
    self.skill_mod: float

  New fields to add:
    self.branch_levels: dict[str, int]   # {"cloud_chaser": 2, "contact_high": 1}
    self.main_branch:   str | None       # set when any branch hits T2; None = uncommitted

  branch_levels defaults to {} for linear skills (never used).
  For branched skills, it is populated with all branch keys at 0 on init.

  New methods:
    can_invest_branch(branch_key: str) -> bool
      Returns True if the player may spend into this branch's next tier.
      Logic:
        if main_branch is None:
          # Not committed yet. Can invest if this branch's level < 1 OR
          # (this branch's level == 1 and the other dipped branch is not yet T1).
          # Simplified: can invest in any branch as long as total branch_levels
          # values sum to <2 (allowing T1+T1 before commit), OR this is already
          # an invested branch going to T2 (commit point).
          current = branch_levels.get(branch_key, 0)
          if current == 0:
            dips_used = sum(1 for v in branch_levels.values() if v > 0)
            return dips_used < 2   # haven't spent 2 dips yet
          elif current == 1:
            return True  # committing T2 into this branch is always allowed
          else:
            return False  # shouldn't reach; guard
        else:
          # Main is committed.
          if branch_key == main_branch:
            return branch_levels[branch_key] < max_main_depth
          else:
            return branch_levels.get(branch_key, 0) < max_dip_depth
              and branch_key not in branches_at_t1  # only one dip allowed
              (refined: only allow dip into a branch not already dipped)

      Cleaner implementation pseudocode:
        current_level = branch_levels.get(branch_key, 0)
        if main_branch == branch_key:
            return current_level < max_main_depth
        if main_branch is not None:
            # Already committed. Dip into one other branch allowed at T1.
            # A dip is valid only if: no other branch (besides main) already
            # has level > 0, OR this branch already has level > 0 (upgrading
            # an existing dip, but max_dip_depth = 1 so no upgrade allowed).
            if current_level >= max_dip_depth:
                return False  # already at dip max
            other_dipped = [k for k, v in branch_levels.items()
                            if k != main_branch and v > 0]
            return len(other_dipped) == 0 or branch_key in other_dipped
        else:
            # No main yet. T1 dip into any branch allowed until 2 branches
            # have T1. T2 commitment into any branch is always allowed.
            if current_level == 0:
                dipped = [k for k, v in branch_levels.items() if v > 0]
                return len(dipped) < 2
            elif current_level == 1:
                return True   # spending T2 commits this branch
            return False

    invest_branch(branch_key: str) -> dict | None
      Increments branch_levels[branch_key] by 1.
      If new level == 2 and main_branch is None, sets main_branch = branch_key.
      Returns the perk dict just unlocked, or None on error.

    total_level(self) -> int
      Returns len(trunk) levels earned + sum(branch_levels.values()).
      Used as a drop-in replacement for self.level in XP-gated checks.
      For linear skills, total_level == self.level.

    get_earned_perks(self) -> list[dict]
      Returns all currently earned perks in order: trunk perks (up to trunk
      level earned), then each branch's perks in the order they were taken.
      Used by _apply_perk dispatch and perk description rendering.

PASSIVE PERK CHECKS IN ENGINE:
  The engine currently checks: "if smoking_level >= 3: trigger_phat_cloud()"
  After branching, checks become branch-aware:

  Before (linear):
    if engine.player_skills["Smoking"].level >= 3:
        trigger_phat_cloud()

  After (branching):
    skill = engine.player_skills["Smoking"]
    if _has_perk(skill, "trunk", 3):           # trunk level 3 = Phat Cloud
        trigger_phat_cloud()
    if _has_branch_perk(skill, "cloud_chaser", 1):  # Thick Cloud
        cloud_target_count = 2
    if _has_branch_perk(skill, "cloud_chaser", 2):  # Hotbox
        cloud_radius = 4  # all in radius

  Helper functions (add to skills.py or engine.py utility section):
    def _has_trunk_perk(skill: Skill, level: int) -> bool:
        """True if this branched skill has trunk level >= level."""
        if isinstance(SKILL_PERKS.get(skill.name), list):
            return skill.level >= level  # linear fallback
        return skill.trunk_level >= level

    def _has_branch_perk(skill: Skill, branch_key: str, tier: int) -> bool:
        """True if branch_key has been invested to >= tier."""
        return skill.branch_levels.get(branch_key, 0) >= tier

    NOTE: trunk_level needs to be tracked separately from branch investment.
    Add self.trunk_level: int = 0 to Skill.__init__, incremented when spending
    into trunk perks. This is unambiguous because trunk perks are always spent
    in order 1-2-3 before branches unlock.

COLUMN EXPLANATIONS:
  Left panel columns: Skill name (18), Lv (3), Pot (5), Branch (10)
  "Lv" = total level (trunk + branch investment sum)
  "Pot" = potential_exp available to convert
  "Branch" = short form: "Cloud T2" (main branch name + tier), "—" for no branch
    or linear skill, "Dip T1" if only a dip taken

RIGHT PANEL — TREE VIEW:
  Updates instantly when cursor moves in left panel. No additional keypress needed.

  For LINEAR skills (not yet branched):
    Shows a flat perk list, one per row, with [+]/[ ] markers.
    No column dividers, no Tab behavior. Identical to current perk display.

  For BRANCHED skills:
    Top section: trunk perks (always L1-L3 for pilot skills)
    Below trunk: column divider showing branches side by side
    Each column header: branch name + ★ if main branch
    Each column row: [+]/[ ]/[X] marker + perk name
    The > cursor appears in front of the currently active branch column's
    next-affordable perk (the one Enter would invest in)

MARKERS:
  [+]  Earned / unlocked. This perk is active.
  [ ]  Next available. Can be spent on. Cursor moves here first on Tab.
  [X]  Locked. Either restriction prevents it, or branch is inaccessible.
       XP-locked (not enough potential_exp) also shows as [ ] with dim color.
  ★   Main branch marker in column header. Set once any branch hits T2.
  >   Spend cursor. Where Enter will invest. Shown on one perk at a time.

DESCRIPTION BOX (bottom-left, 3 lines):
  Shows the perk currently under the > cursor. Name on line 1, description
  wrapped to 40 chars on lines 2-3. If cursor is on a trunk perk during
  linear-mode viewing, shows that perk's description.

NAVIGATION:
  Up/Down:   Move between skills on the left panel. Right panel updates live.
  Tab:       Cycle active branch column (moves > cursor between branch columns).
             For linear skills, Tab does nothing.
             For branched skills with only 2 branches, Tab toggles between them.
             Tab wraps: A → B → C → A (for 3-branch skills).
  Enter:     Invest skill points in whatever perk the > cursor indicates.
             If trunk not yet complete: invests in next trunk level (no branch
             selection needed — trunk is linear).
             If trunk complete: invests in the active branch's next available tier.
             If the investment target has insufficient SP or violates restriction:
             flash error message "Not enough SP" or "Branch locked" in desc box.
  Esc:       Close skills overlay.

KEY PRESS COUNT TARGETS:
  Spend on Smoking Cloud Chaser T1 (first branch invest):
    Navigate to Smoking (Down×N), Tab if needed to Cloud Chaser column, Enter.
    3-4 keys.
  Spend on a linear skill like Jaywalking:
    Navigate to Jaywalking (Down×N), Enter.
    2 keys. Same as today.
  Check Stabbing perk layout:
    Navigate to Stabbing. 1-2 keys. Tree shows immediately.

BRANCH COLUMN WIDTHS:
  2-branch split: each column ~20 chars wide in the right panel
  3-branch split: each column ~14 chars wide — perk names may be abbreviated
    (use short_name field for display if added to perk dict, fallback to name[:12])

CURSOR BEHAVIOR WHEN MAIN IS COMMITTED:
  After main_branch is set, Tab still cycles columns, but [X] columns visually
  dim (render in dark grey). The cursor will skip past [X] columns when Tab-ing
  and land on the main column or the one dip branch (if dip hasn't been used).
  If the dip has been maxed (T1 reached), Tab stays on the main column only.

VIII. NEW EFFECTS NEEDED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

"secondhand_slow" (effects.py)
  Category: debuff
  Duration: 2 turns
  modify_energy_gain hook: return gain - 25 (clamped so min is 10)
  Applied to: all enemies within Chebyshev 3 of player at moment of smoking
  Stacks duration: on_reapply resets duration to 2 (does not stack)
  Display: "Slowed" in enemy status, dim cyan color

"smoke_screen_invis" (effects.py)
  Category: buff
  Duration: 3 turns
  On apply: iterate all enemies; any in CHASING state that had player in LOS →
    transition to WANDERING, clear their last_known_player_pos
  On player action check: if player takes a melee attack action OR uses an
    ability that deals damage → expire this effect, then resolve the action.
    The first attack resolves unimpeded (guaranteed hit before expiry).
  modify_movement hook: player still moves normally (invisibility does not
    grant free moves, just removes detection)
  Engine visibility check: while this effect is active, enemies do not gain
    new CHASING state from spotting the player. LOS checks that would trigger
    alert skip the alert step. Wandering enemies remain wandering.
  Breaks on: melee attack, offensive ability use
  Does NOT break on: movement, item use (drinking, eating), defensive abilities

  Implementation note: requires a flag check in the engine's "enemy spot player"
  LOS evaluation step. Something like:
    if player has "smoke_screen_invis" effect: skip enemy-spots-player logic
  And in the player melee/ability attack resolution:
    if player has "smoke_screen_invis":
        remove effect
        guarantee hit (skip accuracy roll)

IX. BALANCE CONSIDERATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SMOKING TRUNK POWER (carried forward unchanged):
  Phat Cloud at trunk L3: 10 + (tol // 2) damage to 1 enemy per smoke.
  At starting tolerance ~8: deals 14 damage. At tol 14 (with TOL boosts): 17.
  Enemy HP floor 1-2: 10-20 HP. One smoke can down a Tweaker.
  This is appropriate — Smoking is a consumable action, joint supply is limited.

CLOUD CHASER SCALING:
  Thick Cloud (T1): 2-target. Good value for 800 SP, extends trunk value.
  Hotbox (T2): All-in-radius AoE is a large power spike. Requires tolerance
    investment + joint supply + being in range. The 4-tile Chebyshev radius
    means ~24 tiles of coverage at most but rarely are 5+ enemies that close.
    Balanced by joint scarcity — you can't spam it.
  Scorched Lungs (T3): 15+tol formula at tol 12-14: 27-29 damage. Still
    single-hit. Execution at 10% HP is a quality-of-life finish, not OP
    (enemies below 10% HP at ~2-3 HP are near-dead anyway). Primary value
    is the +5 flat damage over base formula.

CONTACT HIGH SCALING:
  Roach Fiend (T1): 30% joint conservation is strong economy but unchanged
    from its previous L4 position. If it proves too strong at T1 (too easy to
    access), reduce to 20% or move to T2.
  Secondhand Smoke (T2): 2-turn slow to 3-tile radius. 2 turns at -25 energy/tick
    on ~5 enemies is meaningful but temporary. Requires being near enemies which
    has its own risk. Appropriate for a T2 cost.
  Smoke Screen (T3): 3 turns of invisibility is powerful. The guaranteed first
    hit on exit is a nice payoff. 3 turns limits abuse — you cannot chain smokes
    for sustained invisibility because smoking a second joint while invisible
    would reset the timer (not extend it; on_reapply should reset, not stack).
    Largest risk: combined with Jaywalking Dash, player can dash in, smoke,
    gain invisibility, smoke again (if Roach Fiend preserved joint), get 3 more
    turns. This is a 2-skill combo with real investment — acceptable.

CONNOISSEUR BRANCH:
  This branch is the weakest in raw power, which is intentional — it serves
  build diversity and Rolling synergy rather than carrying solo. Only pilot
  it if Rolling's strain system is fleshed out enough to make strain joints
  reliably available. Otherwise, hold Connoisseur for a later design pass.

SP COSTS VS EARNING:
  Branch T1 = 800 SP. At book_smarts 8 (typical), conversion rate ~19%.
  Earning 800 real SP requires approximately 4200 potential SP.
  A moderate Smoking session (5-8 joints per floor) earns roughly 150-300 SP
  potential per floor. Reaching T1 is a mid-game investment (floors 2-3).
  T2 at 2000 SP is late-game (floor 4 Crack Den or early Meth Lab).
  T3 at 6000 SP requires significant XP farm — Meth Lab territory.
  This curve feels right: branches are long-term payoffs, not floor 1 options.

X. SYNERGIES AND INTERACTIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SMOKING × ROLLING:
  Rolling's L5 "Rollin' Cloud" triggers Phat Cloud on rolling a joint. With
  Cloud Chaser branching, Rollin' Cloud would hit 2 targets (if Thick Cloud)
  or all in 4 tiles (if Hotbox) — this cross-skill interaction extends Rolling's
  value significantly for Cloud Chaser builds. Rolling's strain perks (L4
  Snickelfritz, L3 Spectral Paper for resource conservation) compound with
  Roach Fiend to create a joint-economy powerhouse.

SMOKING × TOLERANCE STAT:
  Cloud Chaser's damage scales with tolerance (15 + tol at Scorched Lungs).
  Any perk, item, or effect that boosts Tolerance directly amplifies Cloud
  Chaser. The trunk itself grants +4 TOL by L2, and Meth-Head / Alcoholism trees
  also provide TOL. High-TOL builds may naturally want Cloud Chaser as a synergy
  payoff for their stat investment.

SMOKING × JAYWALKING:
  Smoke Screen invisibility pairs with Dash (Jaywalking L2) for repositioning.
  Smoke to go invisible, Dash to a new position, enemies lose track. Highly
  effective but requires dual-tree investment. This is an intended emergent combo.

SMOKING × STATUS EFFECTS:
  Contact High's Secondhand Smoke (slow debuff) stacks with existing slow effects
  from items or strains. If a player also has drugs that apply slow, the Smoking
  slow adds duration/intensity. Check that two slow effects on the same enemy
  cap sensibly (suggest: lowest energy/tick wins, not additive, to prevent
  perma-stunlock via smoke-stacking).

SMOKING × ENEMIES WITH FIRE IMMUNITY:
  If Pyromania's Neva Burn Out (immune to fire) is relevant — Phat Cloud is
  smoke/poison damage, not fire. No conflict. Cloud damage type: "toxic" or
  plain "physical" — recommend physical to avoid needing a new damage type.

SAVE STATE: branch_levels and main_branch must be serialized alongside existing
  Skill fields when/if a save system is added. The current game has no save so
  this is a future concern, but the data model should be designed as serializable
  (no lambdas, no runtime-only references in branch tracking data).

PERMADEATH IMPLICATIONS: Branch commitment is permanent within a run.
  The lock-in mechanic is appropriate for permadeath — the weight of the choice
  matters precisely BECAUSE you cannot undo it. If you commit Cloud Chaser and
  later find yourself joint-starved deep in the Meth Lab, that's a consequence
  of your build, not a bug.

REGRESSION FOR LINEAR SKILLS: All 22+ existing linear skills must continue to
  work identically. The isinstance check ensures no behavior change for unmodified
  skills. Test plan: run existing test_smoking_skill_perks.py and ensure all
  current perk applications still trigger correctly after the data model refactor.
  If tests break after the Skill class gets new fields, it means defaults are
  wrong — trunk_level and branch_levels default values must be set correctly so
  that get_perk(skill_name, level) still returns the right dict for linear skills.

DISPLAY COLLISION: The existing skills menu is 72 chars wide. Widening to 100
  may clip on narrow terminal settings. The 120-char total screen width gives
  10 chars of padding per side for the 100-char overlay — acceptable. If the
  user runs at a smaller screen, fall back to showing only the left panel and
  open a second overlay for tree view (handled via render fallback check).

TAB KEY CONFLICT: Tab is already used in GUN_TARGETING (toggle accurate/fast
  mode). Tab in SKILLS context must only fire when menu_state == SKILLS. The
  existing Tab handler should be gated on menu_state to prevent cross-context
  conflicts. Verify this in input_handler.py before implementing.

BRANCH NAMES IN DISPLAY: Short names for 3-branch skills must fit in ~14-char
  columns. Suggest abbreviations: "Cloud Chaser" → "CLOUD", "Contact High" →
  "CONTACT", "Connoisseur" → "CONNO." — consistent with the mockup above.
